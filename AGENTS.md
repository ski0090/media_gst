# AGENT.md: 개발 에이전트 지침서

## 1. 최우선 참조 원칙
- 모든 기술적 결정은 **[SPEC.md](./SPEC.md)**를 따릅니다.
- 빌드 및 의존성 관련 문제 해결은 **[ENV.md](./ENV.md)**를 참조하십시오.

## 2. 코드 생성 및 수정 시 주의사항
- **의존성 추가**: 
  - 새로운 Rust 크레이트는 `cargo add` 명령어를, Flutter 패키지는 `flutter pub add` 명령어를 사용하여 추가하십시오.
  - 추가 시 `ENV.md`에 명시된 GStreamer 버전과의 호환성을 먼저 고려하십시오.
- **플랫폼별 코드**: 빌드 환경([ENV.md](./ENV.md))에 따라 조건부 컴파일(`#[cfg(target_os = "...")]`)을 적극 활용하여 플랫폼 독립적인 코어를 유지하십시오.
- **CI/CD 고려**: 에이전트는 작성된 코드가 `ENV.md`의 표준 환경에서 빌드 가능한지 항상 검토해야 합니다.


## 3. Zero-Copy 구현 검증
- 코드를 작성할 때 픽셀 데이터를 `Uint8List` 등으로 변환하여 복사하는 로직이 포함되지 않도록 주의하십시오.
- 반드시 `SPEC.md`의 **"2.2 비디오 출력 구성"**에 명시된 플랫폼별 최적화 Sink를 사용하십시오.

## 4. 바인딩 생성 (`generate`) 워크플로우
- **자동 갱신**: 본 프로젝트는 `rust/build.rs`에 `flutter_rust_bridge_codegen::generate()`가 포함되어 있습니다. 따라서 IDE(예: VS Code + `rust-analyzer`)에서 Rust 파일을 저장(`checkOnSave`)하면 Dart 바인딩이 **자동으로 최신화**됩니다.
- **수동 실행**: 예외적인 상황(경로 오류, 명시적 결과 확인 등)에서만 `flutter_rust_bridge_codegen generate` 명령을 수동으로 사용하여 동기화를 강제하십시오.

## 5. 커뮤니케이션 지침
- 개발 도중 `SPEC.md`의 설계와 충돌하는 요구사항이 발생할 경우, 사용자에게 아키텍처 변경의 위험성을 먼저 고지하십시오.
